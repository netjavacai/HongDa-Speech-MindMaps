# 演讲思维导图：让问题找人——智能体时代的供热智力思考

> 场合：天津暖通协会·冬病夏治主题会议
> 时长：30分钟 / 18页
> 演讲人：高振松（天津宏达瑞信）

---

## 核心主线

```
人找问题 ──────→ 问题找人
  │                  │
  │  传统模式        │  智能体时代
  │  被动响应        │  主动发现
  │  人工驱动        │  系统驱动
  └──────────────────┘
```

---

## 一、开场呼应（P1-P6，约10min）

### P1 让问题找人 | 1min
- 自我介绍 + 主题说明
- 核心提问：供热企业能不能从"人找问题"走向"问题找人"？

### P2 冬病夏治 | 2min
- 呼应城管委、燃热中心领导要求
- 关键逻辑：**冬季问题集中暴露，夏季才是根上解决的窗口**
  - 冬天 → 抢修·补救·解释·协调
  - 夏天 → 技改·优化·预案·流程固化

### P3 上工治未病 | 2min
- "上工治未病，不治已病"——《黄帝内经》
- 供热问题信号在爆发前已经出现：
  - 换热站长期不稳 / 小区室温反复投诉 / 管网末端水力失衡 / 设备效率下降 / 客服背后是生产问题
- **目标：让隐患提前暴露，让问题主动找人**

### P4 四类压力 | 2min
| 压力 | 痛点 | 记忆 |
|------|------|------|
| **运行** | 数据多，判断难 | 热/网/站/户/能耗数据散 |
| **服务** | 投诉背后是区域/站点问题 | 一个电话→一片区域 |
| **应急** | 寒潮/故障/舆情→快判快调 | 快速判断·快速调度 |
| **管理** | 人忙·专家少·经验难传承 | 老调度退休，知识带走 |

> 口诀：**运·服·应·管**

### P5 人找问题（过去） | 3min
- 人找系统 → 人查数据 → 人问经验 → 人判断 → 人协调
- 典型场景：小区投诉升高 → 客服接 → 生产查站 → 调度看参数 → 维修看现场 → 领导问原因
- **矛盾：数据在系统里，经验在人脑里，流程在部门之间，问题却在用户那里**

### P6 问题找人（未来方向） | 2min
- 系统主动：发现异常 → 分析原因 → 识别岗位 → 推送建议
- 示例：片区室温低+投诉升+温差异常+压力波动 → 系统主动提示水力失衡/末端循环不足
- **关键词：识别 · 解释 · 分派 · 跟踪**

## 二、第一章：AI智能体（P7-P11，约9min）

### P7 智能体是什么 | 2min

**四个能力**（不是聊天机器人）：

| 序号 | 能力 | 说明 |
|------|------|------|
| 1 | **理解业务语言** | 换热站·供回水·二网平衡·室温投诉·热计量·工单·抢修 |
| 2 | **读取系统数据** | 对接生产·客服·设备·能耗·知识库等系统 |
| 3 | **结合规则分析** | 不是只答概念，要结合运行逻辑给判断 |
| 4 | **推动流程** | 提醒·派单·生成报告·复盘·沉淀知识 |

> 口诀：**理解·读取·分析·推动**

### P8 核心能力 | 3min

**五个价值**：

| 价值 | 说明 | 记忆 |
|------|------|------|
| **看数据** | 多系统数据汇聚到同一问题下 | 一个投诉→看小区+站点+管网+室温+工单 |
| **懂业务** | 区分热源/一网/站点/二网/户内 | 不是"温度低怎么办"，而是哪个环节 |
| **会分析** | 趋势·异常·关联关系 | 投诉↑∧天气变化？室温↓∧供水温度？能耗↑∧调节策略？ |
| **能协同** | 问题推给合适的人 | 生产看运行·客服看用户·维修看现场·管理层看风险 |
| **能沉淀** | 每次处置→企业知识资产 | 经验不再只在人脑里 |

> 口诀：**看·懂·析·协·沉**

### P9 场景一：运行隐患提前识别 | 2min
- **夏治期核心价值**：历史运行+投诉+工单+设备+改造记录 → 提前识别风险点
- 识别对象：投诉集中小区 / 参数异常站点 / 效率下降设备 / 二网水力失衡区域
- **不是替代工程师，而是帮助更快找到重点**

### P10 场景二：客服投诉与生产运行联动 | 2min
- 客服系统 ≠ 生产系统 → 但用户问题跨越两者
- 智能体判断四个维度：
  - 个体问题 vs 区域问题？
  - 短期波动 vs 持续趋势？
  - 户内问题 vs 运行问题？
  - 客服解释 vs 生产排查？
- **投诉不再孤立，问题带着线索找岗位**

### P11 场景三：经验传承与应急辅助 | 2min
- 老师傅经验：什么天气→哪些站波动 / 什么话术→什么现场问题 / 什么参数变化→真异常还是扰动
- 智能体价值：历史类似案例 → 涉及站点 → 处置动作 → 最终结果 → 本次排查方向
- **把老员工经验变成企业可复用能力**
- 应急场景：缩短判断时间，提高处置效率

## 三、第二章：vFlex智能管理小站（P12-P15，约7min）

### P12 从云端走向现场 | 1min
- 大量问题发生在**现场**：换热站·泵·阀·表·传感器·控制柜·边缘设备
- 只有云端智能还不够 → 需要边缘侧能力

### P13 为什么需要边缘小站 | 2min

**四个需要**：

| 需要 | 说明 |
|------|------|
| **现场系统多** | 不同厂家·不同协议·不同年代，数据汇聚难 |
| **网络不稳定** | 关键场景需本地运行·本地判断·本地联动 |
| **响应要即时** | 不能等人查系统后再判断，边缘侧先做初步识别 |
| **数据价值高** | 只上传不清洗，细节价值丢失 |

> 口诀：**系统多·网差·响应急·数据贵**

### P14 小站解决什么 | 3min

**五个解决**：

| 解决 | 说明 | 记忆 |
|------|------|------|
| **接入** | 统一不同设备/协议/系统 | 异构归一 |
| **汇聚** | 温度·压力·流量·能耗·设备状态 → 统一数据底座 | 数据融合 |
| **计算** | 异常识别·趋势判断·规则触发 | 边缘智能 |
| **联动** | 控制·报警·上报·工单协同 | 执行闭环 |
| **自治** | 断网时保留基础判断和本地运行能力 | 断网可控 |

> 口诀：**接·汇·算·联·治**

### P15 一云一边协同 | 2min

```
┌─────────────────────────────────────┐
│           一云一边协同              │
├──────────────┬──────────────────────┤
│  AI 智能体   │  vFlex 智能管理小站  │
├──────────────┼──────────────────────┤
│  理解问题    │  接入现场            │
│  分析原因    │  汇聚数据            │
│  调用知识    │  边缘计算            │
│  生成建议    │  本地联动            │
│  推动流程    │  断网自治            │
├──────────────┼──────────────────────┤
│  云端理解    │  边缘执行            │
│  平台协同    │  小站支撑            │
└──────────────┴──────────────────────┘
```

## 四、第三章：宏达瑞信（P16-P18，约6min）

### P16 行业积淀 | 4min

**三个转变**：

| 从 | 到 | 说明 |
|----|----|------|
| 系统建设 | **能力建设** | 不只建系统，更建发现和解决问题的能力 |
| 数据展示 | **问题洞察** | 不只展示数据，更要洞察数据背后的含义 |
| 人工追问题 | **问题主动找人** | 不靠人追，靠系统主动识别和推送 |

### P17 夏治期建议路径 | 2min

**四步路径**：

```
诊断 ──→ 试点 ──→ 复制 ──→ 沉淀
 │         │         │         │
梳理历史   选片区     固化模型   经验→知识库
识别风险   小范围验证 扩展区域   智能体能力
```

> 口诀：**诊·试·复·沉**

### P18 结尾 | 1min
- 核心：让问题**更早被发现、更快被定位、更准地找到人、更稳地被解决**
- 冬季是战场，夏季是窗口
- 回扣主题：**上工治未病 → 让问题找人**
